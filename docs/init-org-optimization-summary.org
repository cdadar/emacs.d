#+title: init-org.el 优化总结清单
#+author: Hermes Agent
#+date: [2026-04-11]

* 已完成

** use-package 结构收敛
- 将多个分散的 ~(use-package org ...)~ 收敛为单一主块。
- 清理了散落的 ~(with-eval-after-load 'org ...)~，避免 Org 主配置分裂。
- 目前 Org 主配置的职责更清晰：
  - ~:custom~ 放基础变量
  - ~:hook~ 放简单 hook
  - ~:bind~ 放按键绑定
  - ~:config~ 放函数定义与集中初始化调用

** 解决配置互相覆盖问题
- 合并了 ~org-agenda-custom-commands~ 的定义，避免不同位置重复 ~setq~ 造成覆盖。
- 合并了 ~org-capture-templates~ 的定义，避免增量修改后又被整体重写。
- 将 Brain capture 模板并入统一 capture 配置，避免模板丢失。

** 清理未接线/历史遗留代码
- 删除了未真正启用的 dynamic agenda 相关 dead code。
- 去掉了旧式 advice/持久化思路带来的维护负担。

** Hook 与 keymap 整理
- 将反复在 hook 中修改 ~org-mode-map~ 的逻辑移出，避免每次进 buffer 重复做全局 keymap 修改。
- 将截图命令抽成独立函数，并绑定到 ~C-c s c~。
- 修正了 prose 相关逻辑里对 Org major mode 的判断。
- 保留了对 prose 体验的整理，但默认不自动启用 ~prose-mode~。

** Org 显示与导出修复
- 在 ~org-mode~ 中启用窗口内软换行，长 heading 与长行内容会在窗口内自动换行显示。
- 为视觉换行增加了 fringe 标记：~visual-line-fringe-indicators~ 设为 ~(left-curly-arrow right-curly-arrow)~，方便区分软换行与真实换行。
- 将 ~org-export-headline-levels~ 提升到 ~5~，让五级 Org heading 在 LaTeX 导出时继续保持为真实标题（直到 ~subparagraph~），而不是降级成 ~enumerate~ / ~\item~。
- 为 quote 导出增加了 ~cdadar/org-latex-fix-quote-paragraph-spacing~，仅在 LaTeX 导出时处理 ~quote~ 环境里的 ~\\ + 空行~，改写为显式竖向间距，避免 PDF 中空行后的内容向右偏移。

** Clock 显示体验
- 恢复了 clocking 时的 header-line 提示：clock in 时显示当前任务，clock out / cancel 时自动清理。

** 第二梯队懒加载优化
- ~org-roam~ 改成真正按需加载后再启用 autosync，而不是通过启动后的 idle timer 提前触发。
- 这些扩展改成了更合理的按需加载方式：
  - ~grab-mac-link~
  - ~ox-ioslide~
  - ~org-mind-map~
  - ~org-pandoc-import~

** Babel / 导出路径去副作用化
- 移除了启动/加载路径中对 ditaa 和 plantuml 的运行时自动下载。
- 去除了以下历史债模式：
  - ~http://~ 下载
  - ~url-copy-file~
  - ~shell-command unzip -p ...~
- 现在改为只发现和配置已有工具：
  - ditaa: 本地 jar 或 ~DITAA_JAR~
  - plantuml: 系统 ~plantuml~ 命令、 本地 jar 或 ~PLANTUML_JAR~
- 未发现工具时只提示，不再在 init 路径里安装软件。

** 主配置函数化
- 将主 ~use-package org~ 中的大块 ~:config~ 拆成按职责划分的 setup 函数：
  - ~cdadar/org-setup-agenda-window-hooks~
  - ~cdadar/org-setup-clocking~
  - ~cdadar/org-setup-babel~
  - ~cdadar/org-setup-crypt~

* 暂不建议动

** 不建议一次性全量改名旧 ~sanityinc/~ 标识
原因：
- 已有名字分布较广
- 可能影响用户习惯、历史配置和 muscle memory
- 本轮已经明确约束为“只改新整理出来的部分”

如果要做，应单独作为“命名空间迁移”任务处理。

** 不建议把所有逻辑都强行塞进 ~:custom~ / ~:hook~
有些逻辑天然更适合保留在函数和 ~:config~ 里，例如：
- advice
- 复杂 capture 模板生成
- Babel 图表工具发现
- Agenda 跳过逻辑
- 自定义命令定义

过度声明式化会让结构更隐晦，不一定更好维护。

** 不建议恢复运行时自动安装 diagram 依赖
已经移除的做法不建议再加回来：
- 启动时联网下载 jar
- 配置加载时执行 unzip / shell 安装流程
- 用不安全的远程源自动取文件

更稳妥的方式应是：
- 明确安装
- 明确路径
- 明确环境变量

** 不建议短期继续大规模重排整个文件顺序
当前已经完成了几轮结构重构并通过验证。
继续大幅重排会增加：
- diff 噪音
- 回归排查成本
- 未来 blame 难度

如果后续还要重排，建议基于明确目标单独进行，例如：
- “只整理 capture 区块”
- “只整理 roam 区块”
- “只整理 export/babel 区块”
